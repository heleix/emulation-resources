Fun with emulators

We will end this module by introducing you to a few emulation options you can use right now and guide you through solving an example access problem. Let's have some fun with emulators!

Local emulators

Local emulators are the most robust and run on a dedicated machine in your reading room or physical location.

This useful guide from EaaSI's Ethan Gates contains a list of emulators.

We encourage you to explore installing an emulator on your computer, but for simplicity for this module, we will focus on emulation in a browser.

Browser-based emulators

Emulators that run in a browser take advantage of the processing capabilities from each user's device to render the emulator and any emulated content. These emulators have the advantage of being accessible online to users outside of your reading room or physical location. A downside is the potentially significant load passed onto the user, rather than a dedicated machine on-site.

Many emulators that run in browsers are written in Javascript. The Internet Archive's emulator from the previous lesson is JSMESS, a Javascript port of the MESS and MAME emulators.

List of lots of Javascript browser emulators!

Rhizome's article on Cory Arcangel's Bomb Iraq embeds an emulation of the full original Macintosh TV environment for the Hypercard stack. Try it out here.

Exercise: OldWeb.Today

To demonstrate the possibilities of browser-based emulators, we will turn to an exercise using Webrecorder's OldWeb.Today emulation site. This project provides access to multiple emulation environments to emulate past browsers and explore archived websites.

December 31, 2020 marked the official end of life for Adobe Flash on the web, once a popular way to create interactive and dynamic content. Although most websites online today no longer use Flash, viewing archived websites from the WayBack Machine, as well as any straggler websites online still using Flash, requires emulating the Flash plugin no longer available in modern browsers. Many of the emulated browsers on OldWeb.Today contain different versions of Flash.

John Henry Hypercard Stack Emulator

Let's start with a current website with elements that require Flash. The John Henry Hypercard Stack Emulator seeks to emulate a promotional item that accompanied the band They Might Be Giants' album John Henry in 1994. This item was created in the same Hypercard program as Bomb Iraq and can no longer run on modern computers.

Unfortunately, the emulation created in 2014 is dependent on Flash plugins and will no longer display on 2021 browsers. Visiting the page just shows a blank space where the Hypercard stack should be.

With OldWeb.Today, we can emulate the Flash plugin and have fun with They Might Be Giants once more.

  1. Open OldWeb.Today in a new window of the browser of your choice. This module was written in Firefox 86.0.

  2. Paste the URL of the John Henry Hypercard Stack Emulator in the URL box.

    http://tmbg.bryceland.com/JHstack.php

  3. Click the radio button next to "Browse Live Web"

  4. Click "Select a Browser" and choose "Firefox 10" (with Flash 32). Clicking this should launch the emulator.

  5. Wait for emulator to load (this may take several minutes). For best results, leave this page open until the emulator has fully loaded.

    NOTE: This may cause your computer to heat up and make loud fan noises. Remember Javascript emulators in-browser require additional computing power that is pushed to the user's device.

  6. Once fully loaded, the John Henry Hypercard stack should be visible. Click in the emulator to control the mouse. Click on an icon to enjoy some signature catchy pop songs!
